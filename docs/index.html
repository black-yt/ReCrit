Scientific critic reasoning

ReCrit learns when criticism should change an answer.

ReCrit is an inter-turn transition-aware reinforcement learning framework for scientific reasoning. It separates useful correction from harmful sycophancy by rewarding how an answer changes from the Initial solution to the Critic solution.

View Code See Results

From final accuracy to transition awareness.

A critic interaction is not just a final answer. ReCrit measures whether feedback repairs an initially wrong solution, preserves an initially correct one, or causes the model to abandon a correct answer.

Why critic reasoning is hard

Scientific tasks require the model to inspect evidence, not merely follow the user's tone. A helpful critic should trigger grounded re-analysis, while misleading criticism should not overwrite a correct conclusion.

Correction Initial wrong, Critic correct. Reward useful revision.
Robustness Initial correct, Critic correct. Reward stable reasoning.
Boundary Initial wrong, Critic wrong. Apply a mild boundary penalty.
Sycophancy Initial correct, Critic wrong. Penalize harmful answer switching.

A transition-aware RL pipeline.

ReCrit samples Initial and Critic solutions, judges both, maps the correctness transition into a quadrant reward, and updates the policy with GRPO-style grouped advantages.

The training pipeline injects critic feedback with multiple attitudes, evaluates Initial-to-Critic correctness transitions, and uses intra-group advantages to update the policy.

Rollout without tail waiting.

Multi-turn rollout can be slow when a batch waits for the longest generation. Dynamic asynchronous rollout lets completed samples advance immediately and uses tail-adaptive completion to reduce waiting bubbles.

Compared with synchronous rollout, asynchronous scheduling reduces GPU waiting bubbles. Tail-adaptive completion further trims slow tails while preserving valid critic transitions.

Concrete correction behavior.

The qualitative examples show the desired interaction pattern: the critic does not reveal the answer, but asks for verification; ReCrit identifies the weak premise and revises only when the re-analysis supports a change.

ChemBench and EarthSE examples where the Initial solution is plausible but wrong, while the Critic solution performs grounded verification and corrects the final answer.
